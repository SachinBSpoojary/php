Explain Various Scalar Data Types in PHP with Examples
10 Marks
▼
📘 Introduction — What Are Scalar Data Types?

In any programming language, the concept of data types is absolutely fundamental because data types tell the compiler or the interpreter what kind of value a particular variable is going to hold during the execution of a program. PHP, being a dynamically typed and loosely typed scripting language, provides a rich set of data types that allow developers to work with many different kinds of information. Among these, scalar data types are considered the most basic and most commonly used category of data types in PHP.

The word "scalar" comes from mathematics and it simply means a single value — that is, a value that holds only one piece of information at a time. Unlike compound data types (such as arrays and objects) which can store multiple values together as a collection, a scalar data type stores exactly one value. In PHP, there are precisely four scalar data types, and every PHP programmer must understand all four of them thoroughly. These four scalar types are: Integer, Float (also called Double), String, and Boolean. Let us now study each one of them in great detail with clear examples and explanations.

1️⃣ Integer Data Type

Definition: An Integer in PHP is a whole number — a number that does not have any decimal point or fractional part. It can be a positive number, a negative number, or zero. Integers are the simplest numeric data type available in PHP and they are used extensively whenever we need to work with whole numbers such as counting items, representing ages, storing quantities, tracking loop counters, and performing mathematical calculations that do not require decimal precision.

In PHP, integers can be specified in several different number systems or bases. The most common format is decimal (base 10), which is the standard numbering system that we use in our everyday life. However, PHP also supports hexadecimal (base 16) integers which are written with a 0x prefix, octal (base 8) integers written with a leading 0 prefix, and binary (base 2) integers written with a 0b prefix. The maximum size of an integer depends on the platform — on a 64-bit system, PHP can handle integers up to approximately 9.2 quintillion (9,223,372,036,854,775,807).

Key Characteristics
Integers have no decimal point, can be positive or negative, support decimal/octal/hexadecimal/binary notation, and PHP automatically converts an integer to float if it exceeds the maximum size (integer overflow).
<?php
  // Declaring integer variables
  $age = 25;             // Positive decimal integer
  $temperature = -10;      // Negative integer
  $zero = 0;              // Zero is also an integer
  $hex = 0x1A;            // Hexadecimal (equals 26)
  $oct = 075;             // Octal (equals 61)
  $bin = 0b1010;           // Binary (equals 10)

  echo $age;               // Output: 25
  echo gettype($age);       // Output: integer
  echo is_int($age);        // Output: 1 (true)
?>

2️⃣ Float (Double) Data Type

Definition: A Float (which is also known as a Double or a floating-point number) is a number that contains a decimal point or a number that is expressed in exponential (scientific) notation. Floats are essential in PHP whenever we need to perform calculations that require decimal precision — for example, working with monetary values, scientific calculations, percentages, measurements, statistical computations, and any arithmetic that goes beyond whole numbers.

The term "floating point" refers to the fact that the decimal point can "float" to different positions within the number, allowing the representation of both very large numbers and very small fractions. PHP uses the IEEE 754 double precision format for storing floating-point numbers, which provides approximately 14 digits of decimal precision. One very important thing to remember for examinations is that floating-point arithmetic can sometimes produce unexpected results due to the way computers represent decimal fractions in binary — for example, the expression (0.1 + 0.2) may not be exactly equal to 0.3 in PHP due to floating-point rounding errors.

Key Characteristics
Floats contain a decimal point or use exponential notation (e.g., 2.5e3 = 2500). They have approximately 14 digits of precision. PHP provides is_float() function to check if a value is a float.
<?php
  $price = 99.99;          // Standard float
  $scientific = 2.5e3;      // Exponential notation (2500)
  $negative = -15.75;       // Negative float
  $tiny = 7.2e-3;          // Very small number (0.0072)

  echo $price;              // Output: 99.99
  echo gettype($price);      // Output: double
  echo is_float($price);     // Output: 1 (true)
?>

3️⃣ String Data Type

Definition: A String in PHP is a sequence of characters — essentially, it is a piece of text. A string can contain letters, numbers, special characters, whitespace, or any combination of these. Strings are one of the most widely used data types in PHP because web applications constantly deal with text data — displaying user names, processing form inputs, generating HTML content, handling URLs, reading files, working with databases, and much more.

In PHP, strings can be declared in four different ways. The two most common methods are using single quotes (' ') and double quotes (" "). The key difference between them is that double-quoted strings support variable interpolation (meaning PHP will automatically replace variable names inside the string with their values) and escape sequences (such as \n for newline, \t for tab), whereas single-quoted strings treat everything literally except for the escaped single quote (\') and escaped backslash (\\). The other two methods are heredoc syntax (which behaves like double quotes) and nowdoc syntax (which behaves like single quotes), both of which are used for multi-line strings. PHP also provides a vast collection of built-in string functions such as strlen(), strtoupper(), strtolower(), substr(), strpos(), str_replace(), and many more.

<?php
  // Single-quoted string
  $name = 'John Doe';

  // Double-quoted string with variable interpolation
  $greeting = "Hello, $name! Welcome.";

  // String with escape sequences
  $message = "Line 1\nLine 2\tTabbed";

  // String functions
  echo strlen($name);        // Output: 8
  echo strtoupper($name);   // Output: JOHN DOE
  echo $greeting;          // Output: Hello, John Doe! Welcome.
?>

4️⃣ Boolean Data Type

Definition: A Boolean is the simplest data type in PHP. It represents a truth value and can hold only one of two possible values: true or false. Booleans are named after the mathematician George Boole, who is famous for developing Boolean algebra — the mathematical foundation of all digital logic and computer science. In PHP, booleans are primarily used for conditional testing — that is, they are used in if-else statements, while loops, comparison operations, logical operations, and any situation where the program needs to make a decision based on whether a condition is true or false.

PHP performs automatic type casting to boolean in many situations. The following values are considered false when converted to boolean: the integer 0, the float 0.0, an empty string "", the string "0", an empty array [], the special type NULL, and SimpleXML objects created from empty elements. Every other value is considered true, including negative numbers, non-empty strings, and arrays with at least one element. Understanding this type casting behaviour is extremely important because it affects how conditional statements evaluate their conditions in PHP programs.

<?php
  $is_logged_in = true;
  $has_permission = false;

  // Using boolean in conditional statement
  if ($is_logged_in) {
    echo "Welcome! You are logged in.";
  } else {
    echo "Please log in first.";
  }

  // Type casting to boolean
  var_dump((bool)0);         // bool(false)
  var_dump((bool)1);         // bool(true)
  var_dump((bool)"");        // bool(false)
  var_dump((bool)"Hello");   // bool(true)
?>

Explain Miscellaneous Operators in PHP with Examples

📘 Introduction — What Are Miscellaneous Operators?

PHP provides a very extensive and well-organized collection of operators that are used to perform various types of operations on variables and values. While most programmers are familiar with the common categories of operators — such as arithmetic operators (+, -, *, /), assignment operators (=), comparison operators (==, ===, !=), and logical operators (&&, ||, !) — there exists a special group of operators in PHP that do not fit neatly into any of these standard categories. These are collectively referred to as Miscellaneous Operators.

Miscellaneous operators in PHP serve unique and specialized purposes that are not covered by the regular operator categories. They provide powerful shortcuts and capabilities that make PHP code more concise, more robust, and easier to maintain. Understanding these operators is essential for every PHP developer because they are frequently used in real-world applications, especially for error handling, shell command execution, type checking, and handling null values gracefully. PHP provides four main miscellaneous operators: the Execution Operator (backticks), the Error Control Operator (@), the Type Operator (instanceof), and the Null Coalescing Operator (??).

1️⃣ Execution Operator (Backticks ` `)

Definition: The Execution Operator in PHP uses backtick characters (` `) to execute shell commands directly from within a PHP script. Whatever command is placed between the backtick characters is sent to the operating system's command shell for execution, and the output of that command is returned as a string that can be stored in a variable, displayed on screen, or processed further. This operator is functionally equivalent to calling the PHP function shell_exec(), and it provides a convenient shorthand for running system-level commands.

This operator is particularly useful when PHP applications need to interact with the underlying operating system — for example, listing files in a directory, checking system information, running database backup scripts, or executing any other command-line utility. However, it is important to note that using the execution operator requires careful attention to security, because if user input is passed directly into a shell command without proper sanitization, it can lead to a dangerous vulnerability known as command injection. The execution operator is disabled when PHP is running in safe mode or when the shell_exec() function has been disabled for security reasons.

<?php
  // Using backticks to execute a shell command
  $output = `ls -la`;       // Linux: list files
  echo "<pre>$output</pre>";

  // Windows equivalent
  $output = `dir`;          // Windows: list files
  echo $output;
?>
Displays the list of files and directories from the server's file system.
2️⃣ Error Control Operator (@)

Definition: The Error Control Operator in PHP is the at symbol (@). When this operator is placed immediately before any PHP expression, it suppresses (hides) any error messages that would normally be generated by that expression. This means that even if the expression causes an error — such as a warning, a notice, or a fatal error — the error message will not be displayed to the user or recorded in the error log (depending on the configuration).

This operator is commonly used in situations where a developer anticipates that a certain operation might fail and wants to handle the failure gracefully without showing ugly error messages to the end user. A very common use case is when opening a file that might not exist, connecting to a database that might be offline, or dividing by a variable that could potentially be zero. However, it is generally considered bad practice to overuse the error control operator because suppressing errors can make debugging extremely difficult. Errors exist for a reason — they tell us that something is wrong in our code. A better approach is to use proper error handling techniques such as try-catch blocks and conditional checks instead of blindly suppressing errors.

<?php
  // Without @ operator — this would show an error
  // if "nonexistent.txt" does not exist

  // With @ operator — error is suppressed
  $file = @fopen("nonexistent.txt", "r");

  if ($file === false) {
    echo "File could not be opened.";
  } else {
    echo "File opened successfully.";
  }
?>
File could not be opened.
3️⃣ Type Operator (instanceof)

Definition: The instanceof operator is a type-checking operator in PHP that is used to determine whether a given object is an instance (object) of a specific class, or whether it is an instance of a class that inherits from (extends) a specific parent class, or whether it implements a specific interface. This operator returns a boolean value — it returns true if the object is an instance of the specified class or interface, and false otherwise.

The instanceof operator is extremely useful in Object-Oriented Programming (OOP) when working with class hierarchies and polymorphism. For example, if you have a function that receives an object as a parameter, you might want to check what type of object it is before performing certain operations on it. This is especially important when you have multiple classes that inherit from a common parent class, or when you have classes that implement a common interface. The instanceof operator helps ensure type safety in your code and prevents runtime errors that could occur if you try to call a method on an object that does not support it.

<?php
  class Animal {
    public $name;
  }

  class Dog extends Animal {
    public function bark() {
      return "Woof!";
    }
  }

  $myDog = new Dog();

  // Check if $myDog is an instance of Dog
  var_dump($myDog instanceof Dog);    // true
  var_dump($myDog instanceof Animal); // true (inherits)
?>

4️⃣ Null Coalescing Operator (??)

Definition: The Null Coalescing Operator (??) was introduced in PHP 7.0 and it provides an extremely convenient and elegant way to handle situations where a variable might be null or might not even exist at all. This operator takes two operands: it returns the first operand if that operand exists and is not null; otherwise, it returns the second operand as a fallback or default value.

Before the introduction of the null coalescing operator, PHP developers had to write verbose code using the isset() function combined with ternary operators to achieve the same result. For example, the old way would be: $username = isset($_GET['user']) ? $_GET['user'] : 'Guest'; — but with the null coalescing operator, this can be simplified to: $username = $_GET['user'] ?? 'Guest';. This makes the code much shorter, much more readable, and much easier to maintain. Furthermore, the null coalescing operator can be chained — meaning you can provide multiple fallback values, and PHP will return the first non-null value it encounters from left to right.

<?php
  // Basic usage — providing a default value
  $username = $_GET['user'] ?? 'Guest';
  echo $username; // Output: Guest (if 'user' not set)

  // Chaining multiple fallback values
  $color = $_GET['color'] ?? $defaultColor ?? 'blue';
  echo $color; // Output: blue

  // Variable that exists and is not null
  $name = "Alice";
  $result = $name ?? "Unknown";
  echo $result; // Output: Alice
?>

📘 Introduction

As we have already learned, PHP supports scalar data types that can hold only a single value at a time. However, in real-world programming, we frequently need to work with collections of data and complex structures that go far beyond simple single values. This is where Compound Data Types come into the picture. Additionally, PHP provides Special Data Types that represent unusual or unique kinds of values that do not fit into either the scalar or compound categories. Together, compound and special data types give PHP the flexibility and power needed to build sophisticated web applications. PHP has two compound data types — Array and Object — and two special data types — NULL and Resource.

📦 PART A — Compound Data Types

1. Array

Definition: An Array in PHP is a compound data type that can store multiple values in a single variable. Instead of declaring separate variables for each piece of related data, an array allows you to group all of those values together under one variable name and access them using indices (for indexed arrays) or keys (for associative arrays). Arrays are one of the most powerful and most frequently used data structures in PHP, and they are used in nearly every PHP application for storing lists of items, database query results, form data, configuration settings, and much more.

PHP supports three types of arrays: Indexed Arrays (where elements are accessed using numeric indices starting from 0), Associative Arrays (where elements are accessed using named keys that you define), and Multidimensional Arrays (which are arrays containing other arrays, creating a matrix-like structure). Arrays in PHP are extremely flexible — they can hold values of different data types simultaneously, they can grow and shrink dynamically, and PHP provides over 80 built-in functions for sorting, searching, filtering, merging, and manipulating arrays.

<?php
  // Indexed Array
  $fruits = array("Apple", "Banana", "Orange");
  echo $fruits[0]; // Output: Apple
  echo $fruits[1]; // Output: Banana

  // Associative Array
  $student = [
    "name" => "Rahul",
    "age"  => 21,
    "grade" => "A"
  ];
  echo $student["name"]; // Output: Rahul
?>

2. Object

Definition: An Object in PHP is an instance of a class. In Object-Oriented Programming (OOP), a class is like a blueprint or template that defines the properties (data) and methods (functions) that its objects will have. When we create an object from a class, we are creating a specific instance of that blueprint with its own set of data values. Objects are the cornerstone of modern PHP development because they allow developers to organize code into logical, reusable, and maintainable units using the principles of encapsulation, inheritance, and polymorphism.

To create an object in PHP, you must first define a class using the class keyword, then create an instance of that class using the new keyword. The class can contain properties (variables that belong to the object) and methods (functions that belong to the object). Inside a method, you use the special variable $this to refer to the current object and access its properties and methods.

<?php
  class Fruit {
    public $name;
    public $color;

    public function setName($name) {
      $this->name = $name;
    }

    public function getName() {
      return $this->name;
    }
  }

  $apple = new Fruit();
  $apple->setName("Apple");
  echo $apple->getName(); // Output: Apple
?>
Apple
⭐ PART B — Special Data Types

1. NULL

Definition: NULL is a special data type in PHP that represents a variable that has no value at all. It is important to understand that NULL is not the same as zero (0), not the same as an empty string (""), and not the same as false. NULL literally means the absence of any value — the variable exists in memory, but it does not point to any meaningful data. A variable can become NULL in three specific situations: (1) when it has been explicitly assigned the constant NULL, (2) when it has been declared but has not yet been assigned any value, and (3) when it has been unset() — that is, its value has been deliberately destroyed using the unset() function.

PHP provides the is_null() function to check whether a variable is NULL. You can also use the identical comparison operator (===) to compare a variable with NULL. Understanding the NULL data type is important because it helps prevent unexpected behaviour in your programs — for example, performing arithmetic operations on NULL values, or trying to access properties of a NULL object, can lead to warnings and errors.

<?php
  $x = NULL;                  // Explicitly assigned NULL
  var_dump($x);                // Output: NULL
  echo is_null($x);            // Output: 1 (true)

  $y = "Hello";
  unset($y);                  // $y is now NULL/undefined
  var_dump(isset($y));          // Output: bool(false)
?>

2. Resource

Definition: A Resource is a special data type in PHP that holds a reference to an external resource — that is, it acts as a pointer or handle to something that exists outside of PHP's own memory space. The most common examples of resources include database connections (such as a MySQL connection handle), file handles (returned when you open a file using fopen()), image canvases (created by the GD library for image manipulation), and cURL sessions (used for making HTTP requests).

Resources are created and managed by PHP's built-in functions — you cannot directly create a resource variable yourself. When a resource is no longer needed, PHP's garbage collector automatically frees the memory associated with it, although it is considered good practice to explicitly close or free resources when you are done with them (for example, using fclose() for file handles and mysqli_close() for database connections). You can use the is_resource() function to check if a variable is a resource, and the get_resource_type() function to find out what type of resource it is.

<?php
  // Opening a file — returns a resource
  $file = fopen("test.txt", "r");

  echo gettype($file);    // Output: resource
  echo get_resource_type($file); // Output: stream

  // Always close the resource when done
  fclose($file);
?>

📘 PART A — Indexed Arrays

An Indexed Array in PHP is a type of array where each element is stored and accessed using a numeric index (also called a numeric key). The indexing starts from 0 by default — meaning the first element is at index 0, the second element is at index 1, the third at index 2, and so on. Indexed arrays are the most basic and most commonly used form of arrays in PHP, and they are ideal for storing ordered lists of items such as a list of student names, a list of products, a series of scores, or any collection of values where the position (order) of each element matters.

Working with indexed arrays in PHP involves several fundamental operations. These operations form the core of array manipulation and include: initializing (creating) the array, adding new elements to the array, accessing existing elements by their index, modifying elements by assigning new values, removing elements using the unset() function, and iterating (looping) through the array to process each element one by one. Let us study each of these six operations in detail with step-by-step examples.

1Initialization (Creating an Indexed Array)

You can create an indexed array in PHP using the array() function or the shorthand square bracket [ ] syntax (available from PHP 5.4 onwards). Both methods work identically. PHP automatically assigns numeric indices starting from 0 to each element in the order they are provided.

<?php
  // Method 1: Using array() function
  $cars = array("Toyota", "Honda", "Ford", "BMW");

  // Method 2: Using square brackets []
  $fruits = ["Apple", "Banana", "Orange"];
?>
2Adding Elements

You can add new elements to an indexed array either by assigning a value directly to a new index position, or by using the array_push() function which appends one or more elements to the end of the array. When using direct assignment, PHP automatically assigns the next available index if you leave the brackets empty.

<?php
  $cars[4] = "Mercedes";           // Direct index assignment
  array_push($fruits, "Grapes");    // Using array_push()
  $fruits[] = "Mango";              // Auto-index (appends)
?>
3Accessing Elements

You access elements of an indexed array by specifying the numeric index inside square brackets after the array variable name. Remember that the first element is always at index 0.

<?php
  echo $cars[0];   // Output: Toyota
  echo $fruits[1]; // Output: Banana
?>
4Modifying Elements
<?php
  $cars[1] = "Chevrolet"; // Honda is now replaced with Chevrolet
?>
5Removing Elements

The unset() function removes a specific element from the array. Note that unset() does not re-index the remaining elements — the indices stay the same, leaving a gap. To re-index, you can use array_values().

<?php
  unset($cars[2]); // Removes "Ford" at index 2
?>
6Iterating Over the Array

You can loop through all elements of an indexed array using a foreach loop (most common and recommended), a for loop (when you need the index), or a while loop.

<?php
  foreach ($fruits as $fruit) {
    echo $fruit . "<br>";
  }
?>

An Associative Array in PHP is a type of array where each element is stored as a key-value pair. Unlike indexed arrays where elements are identified by numeric indices (0, 1, 2, ...), associative arrays use meaningful string keys that you define yourself to identify and access each element. This makes the code much more readable and self-documenting because the key itself describes what the value represents. For example, instead of remembering that index 0 holds the student's name and index 1 holds the age, you can simply use the keys "name" and "age".

Associative arrays are extensively used in PHP for working with configuration settings, representing database records as key-value pairs, processing HTML form data (since $_POST and $_GET are associative arrays), storing user session data, and any other scenario where labelled data is more meaningful than position-based data. The six fundamental operations for associative arrays are exactly the same as indexed arrays — initialization, adding, accessing, modifying, removing, and iterating — but they use string keys instead of numeric indices.

Explain while and do..while Loops in PHP with Examples
10 Marks

📘 Introduction — What Are Loops?

In programming, there are many situations where you need to execute the same block of code multiple times — for example, printing numbers from 1 to 100, processing each record from a database query, displaying each item in a shopping cart, or repeatedly asking a user for valid input until they provide it. Writing the same code 100 times would be extremely inefficient, error-prone, and impractical. This is exactly why programming languages provide loops — control structures that allow a block of code to be executed repeatedly as long as a specified condition remains true.

PHP provides several types of loops, and two of the most fundamental and widely used are the while loop and the do..while loop. Both of these are condition-controlled loops, meaning they continue to execute as long as a given boolean condition evaluates to true. The critical difference between them lies in when the condition is checked — the while loop is a pre-test loop (entry-controlled loop) that checks the condition before executing the loop body, whereas the do..while loop is a post-test loop (exit-controlled loop) that checks the condition after executing the loop body.

🔁 1. The while Loop (Pre-Test / Entry-Controlled Loop)

Definition: The while loop in PHP is a pre-test loop that evaluates a condition before each iteration of the loop body. If the condition is true, the code inside the loop is executed. After the code is executed, the condition is evaluated again. This process repeats until the condition becomes false, at which point the loop terminates and the program continues with the next statement after the loop. The most important characteristic of the while loop is that if the condition is false from the very beginning (i.e., on the first check), then the loop body will never execute even once. This makes the while loop suitable for situations where you want the loop to execute zero or more times depending on the condition.

Every properly written while loop must have three essential components: (1) an initialization statement that sets the starting value of the loop control variable before the loop begins, (2) a condition that is tested before each iteration to determine whether the loop should continue, and (3) an update statement inside the loop body that modifies the loop control variable so that the condition will eventually become false. If the update statement is missing or incorrect, the loop will run forever — this is called an infinite loop, which is a common programming error that can cause the program to hang or crash.

// Syntax
while (condition) {
  // Code to be executed repeatedly
}
<?php
  // Example: Print numbers from 1 to 5
  $count = 1;            // Step 1: Initialization

  while ($count <= 5) {   // Step 2: Condition check
    echo "Count: $count" . "<br>";
    $count++;           // Step 3: Update (increment)
  }
?>

Iteration 2: $count = 2 → condition (2 <= 5) is TRUE → prints "Count: 2" → $count becomes 3
Iteration 3: $count = 3 → condition (3 <= 5) is TRUE → prints "Count: 3" → $count becomes 4
Iteration 4: $count = 4 → condition (4 <= 5) is TRUE → prints "Count: 4" → $count becomes 5
Iteration 5: $count = 5 → condition (5 <= 5) is TRUE → prints "Count: 5" → $count becomes 6
Check again: $count = 6 → condition (6 <= 5) is FALSE → LOOP ENDS
🔄 2. The do..while Loop (Post-Test / Exit-Controlled Loop)

Definition: The do..while loop in PHP is a post-test loop that evaluates the condition after executing the loop body — not before. This means the code inside the loop will always execute at least once, regardless of whether the condition is true or false, because the condition is only checked after the first execution of the loop body. This is the key difference between do..while and while: the while loop might not execute at all if the condition is false initially, but the do..while loop is guaranteed to execute at least one time.

The do..while loop is particularly useful in situations where you need the loop body to run at least once before checking any condition. A classic real-world example is a menu-driven program where you want to display the menu and get user input at least once, and then repeat the process only if the user chooses to continue. Another common example is input validation — where you ask the user to enter a value, check whether it is valid, and keep asking until valid input is provided. Notice that in the syntax, the do..while loop ends with a semicolon (;) after the closing parenthesis of the while condition — this is one of the very few places in PHP where a control structure requires a semicolon at the end, and forgetting it is a common syntax error.

// Syntax
do {
  // Code to be executed at least once
} while (condition);  // Note the semicolon!
<?php
  // Example: Print numbers from 1 to 5
  $count = 1;

  do {
    echo "Count: $count" . "<br>";
    $count++;
  } while ($count <= 5);
?>
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
<?php
  // Proving do..while executes at least once
  $x = 10;

  // while loop — condition is FALSE, body NEVER runs
  while ($x < 5) {
    echo "while: $x"; // This NEVER prints
    $x++;
  }

  // do..while — body runs ONCE even though condition is FALSE
  $x = 10;
  do {
    echo "do-while: $x"; // This prints ONCE!
    $x++;
  } while ($x < 5);
?>
do-while: 10

Explain Any Four Array Functions in PHP with Examples

📘 Introduction

PHP provides an extraordinarily rich library of built-in array functions — over 80 functions — that allow developers to perform virtually any operation on arrays without having to write complex custom code from scratch. These functions cover a wide range of operations including counting elements, adding and removing elements, sorting arrays, searching for values, merging arrays, extracting portions of arrays, filtering elements, and much more. Understanding the most commonly used array functions is essential for every PHP developer because arrays are the backbone of data storage and manipulation in PHP applications. Let us now study seven of the most important and most frequently used array functions in PHP, each explained with a complete working example.

1️⃣ count() — Count Elements in an Array

The count() function is one of the most basic and most frequently used array functions in PHP. It returns the total number of elements present in an array as an integer value. This function is extremely useful when you need to know how many items are in a list, how many records were returned from a database query, or when you need to use the element count as the upper limit in a for loop. The count() function accepts an optional second parameter called mode — if you pass the constant COUNT_RECURSIVE (or the integer 1), it will count all elements recursively in multidimensional arrays.

<?php
  $numbers = array(1, 2, 3, 4, 5);
  $count = count($numbers);
  echo "Number of elements: $count";
?>
Number of elements: 5
2️⃣ array_push() — Add Elements to End of Array

The array_push() function is used to add (append) one or more new elements to the end of an existing array. It modifies the original array directly (in-place) and returns the new total number of elements in the array after the addition. This function is very commonly used when you are dynamically building a list of items — for example, adding products to a shopping cart, collecting user inputs in a loop, or aggregating results from multiple sources. If you only need to add a single element, the shorthand syntax $array[] = value; is more efficient than calling array_push().

<?php
  $fruits = array("Apple", "Banana", "Orange");
  array_push($fruits, "Grapes", "Watermelon");
  print_r($fruits);
?>
Array ( [0] => Apple [1] => Banana [2] => Orange [3] => Grapes [4] => Watermelon )
3️⃣ array_pop() — Remove Last Element from Array

The array_pop() function removes the last element from an array and returns that removed element. This function modifies the original array by shortening it by one element. If the array is empty or is not an array, it returns NULL. Together with array_push(), the array_pop() function allows arrays to be used as a stack data structure (Last-In-First-Out or LIFO), which is useful in many algorithms and applications such as undo operations, expression evaluation, and backtracking algorithms.

<?php
  $fruits = array("Apple", "Banana", "Orange", "Grapes");
  $last = array_pop($fruits);
  echo "Removed: $last";
  print_r($fruits);
?>
Removed: Grapes
Array ( [0] => Apple [1] => Banana [2] => Orange )
4️⃣ array_reverse() — Reverse Array Order

The array_reverse() function returns a new array with all the elements of the original array arranged in reverse order. The last element becomes the first, the second-to-last becomes the second, and so on. This function does not modify the original array — instead, it creates and returns a completely new reversed array. It accepts an optional boolean parameter called preserve_keys: if set to true, the original numeric keys are preserved; if false (default), the keys are re-indexed starting from 0.

<?php
  $numbers = array(1, 2, 3, 4, 5);
  $reversed = array_reverse($numbers);
  print_r($reversed);
?>
Array ( [0] => 5 [1] => 4 [2] => 3 [3] => 2 [4] => 1 )
5️⃣ array_merge() — Merge Two or More Arrays

The array_merge() function combines two or more arrays into a single array. It appends the values of the second array to the end of the first array, the third array after the second, and so on. If the input arrays have the same string keys, the later value will overwrite the earlier one. However, if the arrays have numeric keys, the values are appended and the keys are re-indexed starting from 0. This function is very useful for combining data from multiple sources.

<?php
  $fruits1 = array("Apple", "Banana");
  $fruits2 = array("Orange", "Grapes");
  $all = array_merge($fruits1, $fruits2);
  print_r($all);
?>
Array ( [0] => Apple [1] => Banana [2] => Orange [3] => Grapes )
6️⃣ array_slice() — Extract a Portion of an Array

The array_slice() function extracts a slice (portion) of an array and returns it as a new array. It takes three main parameters: the array, the offset (starting position), and the length (how many elements to extract). If the offset is negative, the slice starts that many elements from the end of the array. This function is useful for pagination, extracting top N elements, or working with subsets of data.

<?php
  $numbers = array(1, 2, 3, 4, 5);
  $slice = array_slice($numbers, 2, 2);
  print_r($slice);
?>
Array ( [0] => 3 [1] => 4 )
7️⃣ array_search() — Search for a Value in an Array

The array_search() function searches for a specific value within an array and, if found, returns the key (index) of the first matching element. If the value is not found, it returns false. By default, the comparison is loose (==), but you can pass true as the third parameter to enable strict comparison (===), which checks both the value and the data type. This function is extremely useful for finding the position of a specific item in a list.

<?php
  $fruits = array("Apple", "Banana", "Orange", "Grapes");
  $key = array_search("Orange", $fruits);
  echo "Key of 'Orange': $key";
?>
Key of 'Orange': 2
